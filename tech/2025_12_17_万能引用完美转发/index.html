万能引用与完美转发

万能引用

在了解万能引用之前我们需要先理解引用折叠的概念:

  • 引用折叠

    • 假设我们有类型T
    • T& && -> T&
    • T&& && -> T&&

  • 模板匹配:

    template<typename T>
void foo(T&& x);

    当我们使用lvalue实参(比如 int x, char& y, std::string&& z)去调用foo的时候,template会展开为

    void foo((int&) && x) // 对应为int x 因为int只能与T = int& 的模板匹配
void foo((char&) && x) // 对应为 char& y
void foo((std::string&) && x) // 对应为 std::string&& z

    考虑最后为何std::string&& z会展开为std::string& ? 这是因为实质上z为lvalue,在模板类型自动推导的时候T是根据value的类型来推导的，因为std::string&&实际上是可以寻址的变量，位于 = 左侧，所以其实际上为 lvalue ,模板此时展开为std::string&

    此后 int& && 会被折叠为int&, char& && 同理也会被折叠为char &, std::string& && 折叠为 std::string&

    而当调用foo的实参为rvalue时(foo(5), foo(std::move(y)) // long y = 314, foo(rvalue_fun()) //rvalue_fun 的signature: char rvalue_fun(void))

    void foo((int&&) && x)
void foo((long&&) && x) //static_cast<long&&>(y) -> long&&
void foo((char) && x)

    此时int&& && 会被折叠为int&&, 同理long&& && -> long&&, char &&->char&&

    因此，通过模板匹配+引用折叠，我们成功地将lvalue与rvalue区分了出来，这就是万能引用的实现原理，通过万能匹配，我们不仅仅能匹配int, char…等基础类型，还能够匹配int&, 100等，为函数提供了一个通用的入口

完美转发

当我们深入理解万能引用的概念后, 对于完美转发的掌握就信手拈来了, 考虑一个问题, 当我们试图通过构造一个包装器统一处理差异性对象时，我们用最简单的万能引用成功接住了参数:

void sink(std::string&& s)
{
    std::cout << "rvalue\n";
}

void sink(const std::string& s)
{
    std::cout << "lvalue\n";
}

template<typename T>
void wrapper(T&& arg)
{
    sink(arg);
}

但是当调用的时候，我们发现

std::string s = "hello";

wrapper(s);            // 输出 lvalue
wrapper(std::move(s)); // 输出 lvalue

这是为什么呢, 我们std::move(s)不是已经转换为了右值对象了吗? 这是因为,在wrapper函数中,我们虽然接住了传入的参数, 但是在此函数的作用域中, 无论 std::string& arg还是std::string&& arg, 其中的arg都是lvalue,丢失了其原本的信息, 因此, 为了能实现完全地转发参数value属性, 我们引入了完美转发的概念:

template<typename T> //简化版本的forward
constexpr T&& forward(std::remove_reference_t<T>& arg) noexcept
{
    return static_cast<T&&>(arg);
}

当我们输入的参数为U&时,T = U& -> T&& = U&, 输入参数为U&&时, T = U&& -> T&& && = U&&,这样我们依赖于引用折叠，就将返回的函数类型进行了区分, 这样我们就可以在wrapper中区分传入参数的lvalue/rvalue属性，将其放入对应的函数中去, 这就体现出了其转发的含义

template<typename T>
void wrapper(T&& arg)
{
    sink(std::forward<T>(arg)); //使用完美转发
}

这样下来当wrapper的T被推导为U&&时,经过std::forward<U&&>(arg)推导过后,就返回了U&& 类型，与signature 为 void sink(std::string&& s)的函数相匹配, 成功实现了转发

std::string s = "world";

wrapper(s);            // 输出 lvalue
wrapper(std::move(s)); // 输出 rvalue
